Hardening, headers y SSL para producción

Unidad 9.4: Seguridad en Apache

Introducción

Hardening de Apache significa: reducir la superficie de ataque, ocultar información sensible, habilitar headers de seguridad, y configurar SSL/TLS correctamente.

Hardening Básico

1. Ocultar Información del Servidor

BASH
# Crear configuración de seguridad
$ sudo tee /etc/apache2/conf-available/security-hardening.conf <<'EOF'
# Ocultar versión y SO
ServerTokens Prod
ServerSignature Off
ServerResolver Off

# DisableTRACE (previene XST)
TraceEnable Off

# Timeout reducido (default es 300)
Timeout 30

# KeepAlive configuration
KeepAlive On
MaxKeepAliveRequests 100
KeepAliveTimeout 5
EOF

# Habilitar configuración
$ sudo a2enconf security-hardening

# Validar y recargar
$ sudo apachectl configtest && sudo systemctl reload apache2
1
ServerTokens Prod: Solo muestra “Apache”, sin versión

Verificar Hardening Básico

BASH
# Antes
$ curl -I http://localhost | grep Server
Server: Apache/2.4.58 (Ubuntu)

# Después
$ curl -I http://localhost | grep Server
Server: Apache
1
ServerTokens Prod: Oculta versión y SO

Security Headers

Headers Esenciales

BASH
# Habilitar módulo headers
$ sudo a2enmod headers

# Crear configuración de headers
$ sudo tee /etc/apache2/conf-available/security-headers.conf <<'EOF'
<IfModule mod_headers.c>
    # Prevenir MIME sniffing
    Header always set X-Content-Type-Options "nosniff"
    
    # Prevenir clickjacking
    Header always set X-Frame-Options "SAMEORIGIN"
    
    # Referrer policy seguro
    Header always set Referrer-Policy "strict-origin-when-cross-origin"
    
    # XSS protection (legacy pero útil)
    Header always set X-XSS-Protection "1; mode=block"
    
    # Ocultar versión de PHP u otros
    Header always set X-Powered-By "Production"
    
    # Permissions Policy (navegador)
    Header always set Permissions-Policy "geolocation=(), microphone=(), camera=()"
</IfModule>
EOF

# Habilitar configuración
$ sudo a2enconf security-headers

# Validar y recargar
$ sudo apachectl configtest && sudo systemctl reload apache2
1
Headers de seguridad: Los 5 que todo servidor debe tener

Verificar Headers

BASH
# Ver todos los headers
$ curl -I http://localhost
HTTP/1.1 200 OK
Server: Apache
X-Content-Type-Options: nosniff
X-Frame-Options: SAMEORIGIN
Referrer-Policy: strict-origin-when-cross-origin
X-XSS-Protection: 1; mode=block
X-Powered-By: Production
Permissions-Policy: geolocation=(), microphone=(), camera=()
1
Todos los headers: Verificación de hardenización

Content Security Policy (CSP)

BASH
# CSP básico - ajustar según la aplicación
$ sudo tee /etc/apache2/conf-available/csp-basic.conf <<'EOF'
<IfModule mod_headers.c>
    # CSP restrictiva (ajustar según necesidad)
    Header always set Content-Security-Policy "\
        default-src 'self'; \
        script-src 'self' 'unsafe-inline' 'unsafe-eval'; \
        style-src 'self' 'unsafe-inline'; \
        img-src 'self' data: https:; \
        font-src 'self' data:; \
        connect-src 'self'; \
        frame-ancestors 'self'; \
        base-uri 'self'; \
        form-action 'self'; \
        frame-ancestors 'none' \
    "
</IfModule>
EOF
1
CSP: Content Security Policy - protege contra XSS e injection

SSL/TLS en Producción

Instalar SSL

BASH
# Instalar certbot
$ sudo apt install -y certbot python3-certbot-apache

# Habilitar módulo SSL
$ sudo a2enmod ssl

# Habilitar sitio SSL default
$ sudo a2ensite default-ssl

# Ver configuración
$ cat /etc/apache2/sites-available/default-ssl.conf | grep -A5 "SSLEngine"
SSLEngine on
SSLCertificateFile      /etc/ssl/certs/ssl-cert-snakeoil.pem
SSLCertificateKeyFile   /etc/ssl/private/ssl-cert-snakeoil.key
2
default-ssl: Plantilla SSL básica

Configurar SSL Seguro

BASH
# Crear configuración SSL profesional
$ sudo tee /etc/apache2/conf-available/ssl-secure.conf <<'EOF'
<IfModule mod_ssl.c>
    # SSL Protocol versions (disable SSLv3, TLS 1.0, 1.1)
    SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1
    
    # Ciphers seguros
    SSLCipherSuite ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384
    SSLHonorCipherOrder off
    
    # SSL options
    SSLOptions +StdEnvVars +ExportCertData
    
    # HSTS (HTTP Strict Transport Security)
    Header always set Strict-Transport-Security "max-age=63072000; includeSubDomains; preload"
    
    # OCSP Stapling
    SSLUseStapling off
    SSLStaplingCache "shmcb:logs/ssl_stapling(32768)"
</IfModule>
EOF

$ sudo a2enconf ssl-secure
$ sudo apachectl configtest && sudo systemctl reload apache2
1
SSL seguro: TLS 1.2/1.3, ciphers modernos, HSTS

Rate Limiting

Configuración de Rate Limit

BASH
# Habilitar módulo
$ sudo a2enmod ratelimit

# Crear configuración
$ sudo tee /etc/apache2/conf-available/rate-limit.conf <<'EOF'
<IfModule mod_ratelimit.c>
    # 100 requests por segundo por IP
    ServerRateLimitRPS 100
    
    # Burst de hasta 200
    ServerRateLimitBurst 200
    
    # Bytes por segundo
    ServerRateLimitAvgRate 100
    ServerRateLimitBurst 500
</IfModule>
EOF

$ sudo a2enconf rate-limit
$ sudo apachectl configtest && sudo systemctl reload apache2
1
Rate limiting: Protege contra DoS básico

Protección de Archivos

.htaccess

BASH
# Proteger archivos sensibles
$ sudo tee /var/www/html/.htaccess <<'EOF'
# Denegar acceso a todos los archivos
<FilesMatch "^\.(htaccess|htpasswd|git|env|config|ini|xml|json|md|yml)$">
    Require all denied
</FilesMatch>

# Prevenir directory listing
Options -Indexes

# Prevenir acceso a directorios
<Directory />
    Require all denied
</Directory>

# Permitir solo archivos específicos
<FilesMatch "\.(jpg|jpeg|png|gif|ico|css|js|html|php)$">
    Require all granted
</FilesMatch>
EOF
1
.htaccess: Reglas de acceso por directorio

AllowOverride

BASH
# Enable .htaccess en VirtualHost
$ sudo sed -i 's/AllowOverride None/AllowOverride All/' /etc/apache2/sites-available/mysite.conf

$ sudo apachectl configtest && sudo systemctl reload apache2

Autenticación Básica

# Crear archivo de passwords
$ sudo htpasswd -c /var/www/clients/mysite/.htpasswd admin

# Agregar más usuarios
$ sudo htpasswd /var/www/clients/mysite/.htpasswd developer

# Crear .htaccess
$ sudo tee /var/www/clients/mysite/public/.htaccess <<'EOF'
AuthType Basic

Casos de Estudio

Caso 1: Servidor Expuesto a Internet con Scanner

Escenario: Servidor recibiendo miles de requests a paths aleatorios.

BASH
# Ver logs
$ sudo tail -100 /var/log/apache2/access.log
192.168.1.100 - - [31/Mar/2026:10:00:00] "GET /wp-admin/ HTTP/1.1" 404
192.168.1.100 - - [31/Mar/2026:10:00:01] "GET /phpmyadmin HTTP/1.1" 404
192.168.1.100 - - [31/Mar/2026:10:00:02] "GET /.git/config HTTP/1.1" 404

Solución:

BASH
# 1. Rate limiting
$ sudo a2enmod ratelimit
$ sudo a2enconf rate-limit

# 2. Bloquear user-agents sospechosos
$ sudo tee /etc/apache2/conf-available/block-bots.conf <<'EOF'
<IfModule mod_setenvif.no>
    SetEnvIfNoCase User-Agent "Nikto|sqlmap|nessus|Nmap|masscan|metasploit" bad_bot
    <Limit GET POST>
        Order allow,deny
        Allow from all
        Deny from env=bad_bot
    </Limit>
</IfModule>
EOF

$ sudo a2enconf block-bots
$ sudo systemctl reload apache2
  1. Scanner: Buscando vulnerabilidades comunes

Caso 2: Need HTTPS pero no Certbot

Escenario: Certificado auto-firmado para desarrollo interno.

BASH
# Generar certificado auto-firmado
$ sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/ssl/private/apache-selfsigned.key \
    -out /etc/ssl/certs/apache-selfsigned.crt \
    -subj "/C=US/ST=State/L=City/O=Org/CN=internal.example.com"
1

# Configurar VirtualHost con SSL
$ sudo tee /etc/apache2/sites-available/internal-ssl.conf <<'EOF'
<VirtualHost *:443>
    ServerName internal.example.com
    DocumentRoot /var/www/internal/public
    
    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/apache-selfsigned.crt
    SSLCertificateKeyFile /etc/ssl/private/apache-selfsigned.key
    
    <Directory /var/www/internal/public>
        Options -Indexes +FollowSymLinks
        AllowOverride None
        Require all granted
    </Directory>
</VirtualHost>
EOF

$ sudo a2ensite internal-ssl.conf
$ sudo apachectl configtest && sudo systemctl reload apache2

# Probar (aceptar certificado)
$ curl -k https://localhost
1
Certificado auto-firmado: Para testing interno
2
-k: Aceptar certificados no válidos

Caso 3: Content Leaking

Escenario: Archivos sensibles accesibles desde web.

BASH
# Ver qué es accesible
$ curl http://localhost/.env
DB_PASSWORD=secret123
1

# Proteger
$ sudo tee /etc/apache2/conf-available/prevent-leak.conf <<'EOF'
<FilesMatch "^\.(env|git|htaccess|htpasswd|ini|conf|config|yml|yaml|json|md|sql)$">
    Require all denied
</FilesMatch>

# Also prevent directory listing
<DirectoryMatch "^/var/www/.*/">
    Options -Indexes
</DirectoryMatch>
EOF

$ sudo a2enconf prevent-leak
$ sudo systemctl reload apache2

# Verificar
$ curl -I http://localhost/.env
HTTP/1.1 403 Forbidden
1
.env expuesto: Credentials en texto plano
2
403 Forbidden: Archivo bloqueado

Checklist de Seguridad Apache

Config Valor Recomendado Estado
ServerTokens Prod ⏳
ServerSignature Off ⏳
TraceEnable Off ⏳
X-Frame-Options SAMEORIGIN ⏳
X-Content-Type-Options nosniff ⏳
Referrer-Policy strict-origin-when-cross-origin ⏳
HSTS (SSL) max-age=63072000 ⏳
SSLProtocol TLSv1.2/TLSv1.3 only ⏳
Rate Limit 100 RPS ⏳
AllowOverride None (o configuración específica) ⏳

Resumen

Tema Comando Clave Propósito
Hardening ServerTokens Prod Ocultar información
Headers mod_headers Seguridad HTTP
SSL mod_ssl + certbot HTTPS
Rate Limit mod_ratelimit Prevenir DoS
Auth htpasswd Autenticación
Files <FilesMatch> Bloquear archivos
