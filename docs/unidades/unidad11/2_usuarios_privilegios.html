Unidad 11.2: Usuarios, Privilegios y Seguridad

Introducción

La seguridad en bases de datos se basa en: usuarios por aplicación, privilegios mínimos, y acceso restringido por host. En esta unidad vas a aprender a crear usuarios correctamente, gestionar privilegios, y implementar políticas de contraseña seguras.

Arquitectura de Privilegios

Sistema de Privilegios MariaDB

┌─────────────────────────────────────────────────────────────────┐
│              SISTEMA DE PRIVILEGIOS MARIADB                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   USUARIO                    PRIVILEGIOS                        │
│  ┌─────────┐               ┌──────────┐                        │
│  │user@host│ ───────────> │ privileges│                        │
│  │         │               │          │                        │
│  └─────────┘               │ SELECT   │ → Leer datos          │
│                            │ INSERT   │ → Insertar datos      │
│                            │ UPDATE   │ → Actualizar datos    │
│                            │ DELETE   │ → Eliminar datos      │
│                            │ CREATE   │ → Crear objetos       │
│                            │ DROP     │ → Eliminar objetos    │
│                            │ GRANT    → → Dar privilegios    │
│                            └──────────┘                        │
│                                                                  │
│   FLUSH PRIVILEGES → Aplicar cambios sin reiniciar               │
└─────────────────────────────────────────────────────────────────┘

Crear Base de Datos

Crear DB con Charset Correcto

BASH
$ sudo mariadb

# Crear base de datos con charset utf8mb4
CREATE DATABASE appdb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
1

# Verificar
SHOW CREATE DATABASE appdb;
1
utf8mb4: Soporte completo para Unicode (emojis, etc.)
2
Collation unicode_ci: Case-insensitive para español

Tablas con Charset Correcto

BASH
USE appdb;

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
1
InnoDB + utf8mb4: Configuración recomendada

Crear y Gestionar Usuarios

Usuario para Aplicación

BASH
# Crear usuario con password seguro
CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'AppPass-ChangeMe-2026!';
1

# Otorgar privilegios específicos
GRANT SELECT, INSERT, UPDATE, DELETE ON appdb.* TO 'appuser'@'localhost';
FLUSH PRIVILEGES;
1
Password seguro: Mínimo 16 caracteres, mayúsculas, minúsculas, números, símbolos
2
Privilegios mínimos: Solo lo que necesita la aplicación

Verificar Acceso

BASH
# Probar conexión como el nuevo usuario
$ mariadb -u appuser -p appdb -e 'SHOW TABLES;'
1

# Ver qué puede hacer
SHOW GRANTS FOR 'appuser'@'localhost';
1
Conexión exitosa: Puede conectar y ver su DB
2
GRANTS: Muestra privilegios del usuario

Tipos de Usuarios por Host

BASH
# Usuario solo localhost (aplicación en mismo servidor)
CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'Password123!';

# Usuario desde red local (servidor web en otra máquina)
CREATE USER 'appuser'@'10.0.1.0/255.255.255.0' IDENTIFIED BY 'Password123!';

# Usuario desde cualquier lado (NO recomendado para producción)
CREATE USER 'appuser'@'%' IDENTIFIED BY 'Password123!';

# Usuario específico por IP
CREATE USER 'appuser'@'10.0.1.20' IDENTIFIED BY 'Password123!';
1
localhost vs IP vs %: Diferentes niveles de acceso

Privilegios

Niveles de Privilegios

Nivel Alcance Ejemplo
Global Todas las DB GRANT ALL ON *.*
Database Una DB específica GRANT ALL ON appdb.*
Table Una tabla GRANT ALL ON appdb.users
Column Columnas específicas GRANT SELECT(id, name) ON appdb.users
Routine Procedimientos/funciones GRANT EXECUTE ON appdb.*

Privilegios Comunes

BASH
# Aplicación web típica
GRANT SELECT, INSERT, UPDATE, DELETE ON appdb.* TO 'appuser'@'localhost';

# Usuario de solo lectura (reporting)
GRANT SELECT ON appdb.* TO 'readonly'@'%';

# Usuario con DDL (schema changes)
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, DROP ON appdb.* TO 'devuser'@'localhost';

# Usuario administrador (todas las DB)
GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost' WITH GRANT OPTION;
1
GRANT OPTION: Puede dar privilegios a otros usuarios

Revocar Privilegios

BASH
# Quitar privilegio específico
REVOKE DELETE ON appdb.* FROM 'appuser'@'localhost';

# Quitar todos los privilegios
REVOKE ALL PRIVILEGES ON appdb.* FROM 'appuser'@'localhost';

FLUSH PRIVILEGES;
1
REVOKE: Quitar privilegios otorgados

Contraseñas Seguras

Generar Contraseña Segura

BASH
# Con openssl (32 bytes = ~43 caracteres en base64)
$ openssl rand -base64 32
1

# Con pwgen (32 caracteres, segura)
$ pwgen -s 32 1
2

# Con MariaDB (validate_password plugin)
INSTALL PLUGIN validate_password SONAME 'validate_password.so';
SET GLOBAL validate_password_length = 16;
SET GLOBAL validate_password_mixed_case_count = 1;
SET GLOBAL validate_password_number_count = 1;
SET GLOBAL validate_password_special_char_count = 1;
1
openssl: Método nativo del sistema
2
pwgen: Generador dedicado
3
validate_password: Plugin de MariaDB

Políticas de Contraseña

BASH
# Configurar política de contraseñas
$ sudo mariadb

# Ver plugins de contraseña
SHOW PLUGINS LIKE 'validate%';

# Configurar políticas
SET GLOBAL validate_password_policy = 'STRONG';
SET GLOBAL validate_password_length = 20;
SET GLOBAL validate_password_mixed_case_count = 2;
SET GLOBAL validate_password_number_count = 2;
SET GLOBAL validate_password_special_char_count = 2;

# O en my.cnf:
# [mysqld]
# validate_password_policy=STRONG
# validate_password_length=20
1
validate_password: Fuerza contraseñas seguras

Roles (MariaDB 10.2+)

Crear y Asignar Roles

BASH
# Crear rol de lectura
CREATE ROLE 'app_readonly';
GRANT SELECT ON appdb.* TO 'app_readonly';

# Crear rol de escritura
CREATE ROLE 'app_write';
GRANT SELECT, INSERT, UPDATE, DELETE ON appdb.* TO 'app_write';

# Crear rol de admin
CREATE ROLE 'app_admin';
GRANT ALL PRIVILEGES ON appdb.* TO 'app_admin';

# Asignar rol a usuario
GRANT 'app_readonly' TO 'developer'@'localhost';

# Definir rol por defecto
SET DEFAULT ROLE 'app_readonly' FOR 'developer'@'localhost';

FLUSH PRIVILEGES;
1
Roles: Facilitan gestión de privilegios

Usar Roles

BASH
# Usuario activa rol
SET ROLE 'app_write';

# Ver rol activo
SELECT CURRENT_ROLE();

# Cambiar a rol diferente
SET ROLE 'app_readonly';

# Quitar rol activo
SET ROLE NONE;

Gestión de Usuarios en Producción

Auditoría de Usuarios

BASH
# Listar todos los usuarios
SELECT User, Host, plugin, password_lifetime FROM mysql.user ORDER BY User;
1

# Ver privilegios de cada usuario
SELECT user, host, db, select_priv, insert_priv, update_priv, delete_priv 
FROM mysql.db ORDER BY user, db;

# Ver usuarios sin password o expirados
SELECT user, host FROM mysql.user WHERE password = '' OR password IS NULL;
1
Usuarios del sistema: Ver todos
2
Sin password: Usuarios vulnerables

Eliminar Usuarios

BASH
# Eliminar usuario específico
DROP USER 'olduser'@'localhost';

# Eliminar todos los usuarios de un host
DROP USER 'appuser'@'%';

# Verificar
SELECT User, Host FROM mysql.user;
1
DROP USER: Eliminar permanentemente

Lock de Usuarios

BASH
# Bloquear usuario (sin eliminar)
ALTER USER 'olduser'@'localhost' ACCOUNT LOCK;

# Desbloquear
ALTER USER 'olduser'@'localhost' ACCOUNT UNLOCK;

# Cambiar password sin saber la anterior (como root)
SET PASSWORD FOR 'appuser'@'localhost' = PASSWORD('NewPassword123!');
FLUSH PRIVILEGES;
1
ACCOUNT LOCK: useful para empleados que se van

Casos de Estudio

Caso 1: Aplicación No Puede Conectar

Problema:

BASH
$ mariadb -u appuser -p -e "SELECT * FROM users"
ERROR 1045 (28000): Access denied for user 'appuser'@'localhost' (using password: YES)

Diagnóstico:

BASH
# Verificar que el usuario existe
SELECT User, Host, plugin FROM mysql.user WHERE User='appuser';

# Verificar privilegios
SHOW GRANTS FOR 'appuser'@'localhost';

Solución:

BASH
# Resetear password
SET PASSWORD FOR 'appuser'@'localhost' = PASSWORD('NewSecurePassword123!');
FLUSH PRIVILEGES;

Caso 2: Usuario con Demasiados Privilegios

Problema: Usuario de aplicación tiene acceso a todas las bases de datos.

Diagnóstico:

BASH
SHOW GRANTS FOR 'appuser'@'localhost';
# Grants for appuser@localhost: GRANT SELECT, INSERT, UPDATE, DELETE ON *.* TO 'appuser'@'localhost'
1
Con .: Tiene acceso a TODAS las bases de datos

Solución:

BASH
# Revocar privilegios globales
REVOKE SELECT, INSERT, UPDATE, DELETE ON *.* FROM 'appuser'@'localhost';

# Otorgar solo a la DB específica
GRANT SELECT, INSERT, UPDATE, DELETE ON appdb.* TO 'appuser'@'localhost';
FLUSH PRIVILEGES;

Caso 3: Necesito Ver Password de Usuario

Problema: No se puede ver password de usuario (está hasheado).

Solución: No se puede ver el password original. Solo se puede:

BASH
# Resetear password
ALTER USER 'appuser'@'localhost' IDENTIFIED BY 'NewPassword123!';

# O verificar que el password funciona (sin saber el original)
SET PASSWORD FOR 'appuser'@'localhost' = PASSWORD('TestPassword123!');

Resumen

Tema Comando Clave Propósito
Crear DB CREATE DATABASE db CHARACTER SET utf8mb4 Base de datos
Crear Usuario CREATE USER 'user'@'host' IDENTIFIED BY 'pass' Usuario
Privilegios GRANT SELECT ON db.* TO 'user'@'host' Acceso
Roles CREATE ROLE 'rolename' Agrupar privilegios
Auditar SHOW GRANTS FOR 'user'@'host' Ver permisos
