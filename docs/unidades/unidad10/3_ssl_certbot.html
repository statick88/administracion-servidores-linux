Unidad 10.3: SSL/TLS con Certbot

Introducción

Let’s Encrypt ha revolucionado el SSL con certificados gratuitos y automáticos. Certbot es la herramienta oficial que automatiza la emisión, instalación y renovación de certificados TLS.

Fundamentos de SSL/TLS

¿Por qué SSL?

┌─────────────────────────────────────────────────────────────────┐
│                    SSL/TLS EN PRODUCCIÓN                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   SIN SSL                    CON SSL                            │
│  ┌─────────┐               ┌─────────┐                         │
│  │HTTP     │               │HTTPS    │                         │
│  │         │               │         │                         │
│  │User: admin             │User: admin                          │
│  │Pass: secret123   →   │Pass: *** (cifrado)                  │
│  └─────────┘               └─────────┘                         │
│                                                                  │
│   - Datos en texto plano        - Cifrado AES-256               │
│   - Vulnerable a sniffers      - Autenticación del servidor     │
│   - Sin validación identidad   - Certificados verificables     │
│   - Navegadores warn          - Verde en navegador             │
└─────────────────────────────────────────────────────────────────┘

Protocolos SSL/TLS

Protocolo Seguridad Estado
SSLv3 Inseguro ❌ Deprecated
TLS 1.0 Inseguro ❌ Deprecated
TLS 1.1 Inseguro ❌ Deprecated
TLS 1.2 Seguro ✅ Recomendado
TLS 1.3 Muy seguro ✅ Recomendado

Obtener Certificado

Requisitos Previos

Warning⚠️ ADVERTENCIA CRÍTICA

Para obtener un certificado real, el servidor debe ser accesible públicamente.

Lo que podría salir mal: - DNS no apunta al servidor - Firewall bloquea puertos 80/443 - Servidor en red local sin port-forward

Cómo prevenirlo: 1. Verificar DNS: dig +short tu-dominio.com 2. Abrir puertos: sudo ufw allow 80/tcp && sudo ufw allow 443/tcp 3. Probar HTTP público antes de certbot

BASH
# 1. Verificar DNS
$ dig +short example.com
203.0.113.10
1

# 2. Abrir firewall
$ sudo ufw allow 80/tcp
$ sudo ufw allow 443/tcp
$ sudo ufw status

# 3. Probar HTTP público
$ curl -I http://example.com
HTTP/1.1 200 OK
1
DNS: Dominio apunta a IP del servidor
2
HTTP: Respondiendo públicamente

Configuración Generada

Archivos de Certificado

BASH
# Ver estructura
$ sudo ls -la /etc/letsencrypt/live/example.com/
-rw-r--r-- 1 root root 0 Mar 31 10:00 cert.pem       # Certificado
-rw-r--r-- 1 root root 0 Mar 31 10:00 chain.pem      # Chain
-rw-r--r-- 1 root root 0 Mar 31 10:00 fullchain.pem  # cert + chain
-rw-r--r-- 1 root root 0 Mar 31 10:00 privkey.pem    # Clave privada
1
Archivos SSL: Cert, chain, fullchain, privkey

Configuración Nginx

BASH
# Ver configuración creada
$ cat /etc/nginx/sites-available/default-ssl.conf
NGINX
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name example.com www.example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;

    # Redirect HTTP to HTTPS
    if ($scheme != "https") {
        return 301 https://$host$request_uri;
    }
}

SSL Manual (Sin Certbot)

Generar Certificado Self-Signed

BASH
# Generar certificado auto-firmado
$ sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/ssl/private/nginx-selfsigned.key \
    -out /etc/ssl/certs/nginx-selfsigned.crt \
    -subj "/C=US/ST=Test/L=Test/O=Org/CN=localhost"
1
Self-signed: Para testing interno

Configuración SSL Manual

BASH
$ sudo tee /etc/nginx/snippets/ssl-params.conf <<'EOF'
# SSL Certificate
ssl_certificate /etc/ssl/certs/nginx-selfsigned.crt;
ssl_certificate_key /etc/ssl/private/nginx-selfsigned.key;

# SSL Protocolos (TLS 1.2 y 1.3)
ssl_protocols TLSv1.2 TLSv1.3;

# Ciphers seguros
ssl_prefer_server_ciphers off;
ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;

# HSTS
add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload";

# OCSP Stapling (optional)
ssl_stapling on;
ssl_stapling_verify on;
EOF
1
SSL seguro: Configuración manual

Usar en Server Block

BASH
$ sudo tee /etc/nginx/sites-available/ssl-manual.conf <<'EOF'
server {
    listen 443 ssl http2;
    server_name example.com;
    
    include /etc/nginx/snippets/ssl-params.conf;
    
    root /var/www/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
EOF
1
Usar snippet: Configuración SSL manual

Casos de Estudio

Caso 1: Nginx con Nuevo Dominio

Escenario: Agregar SSL a nuevo dominio.

BASH
# 1. DNS apuntando
$ dig +short newdomain.com
203.0.113.10

# 2. Instalar Nginx si no existe
$ sudo apt install -y nginx

# 3. Crear server block
$ sudo tee /etc/nginx/sites-available/newdomain.com <<'EOF'
server {
    listen 80;
    server_name newdomain.com www.newdomain.com;
    root /var/www/newdomain.com;
    index index.html;
}
EOF

$ sudo ln -sf /etc/nginx/sites-available/newdomain.com /etc/nginx/sites-enabled/
$ sudo nginx -t && sudo systemctl reload nginx

# 4. Obtener certificado
$ sudo certbot --nginx -d newdomain.com -d www.newdomain.com --redirect
2
Certbot: Obtener y configurar SSL

Caso 2: Renovar Certificado Expirado

Problema: Certificado expirado, sitio mostrando advertencia.

BASH
# Ver estado
$ sudo certbot certificates
Certificate Name: example.com
    Expiry Date: 2024-03-01 10:00:00+00:00 (EXPIRED)
1

# Renovar
$ sudo certbot renew
2

# Recargar Nginx
$ sudo systemctl reload nginx
1
EXPIRED: Certificado vencido
2
renew: Renovar manualmente

Caso 3: Múltiples Subdominios

Escenario: *.example.com con wildcard o múltiples certificados.

BASH
# Opción 1: Un certificado con múltiples dominios
$ sudo certbot --nginx -d example.com -d www.example.com -d blog.example.com -d shop.example.com

# Opción 2: Certificado wildcard (requiere DNS validation)
$ sudo certbot certonly --manual --preferred-challenges dns -d *.example.com
