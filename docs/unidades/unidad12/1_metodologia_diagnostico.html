Unidad 12: Troubleshooting en Producción

Introducción

El troubleshooting no es “probar cosas al azar”. Es un proceso sistemático: observar, recolectar evidencias, aislar la causa, y validar el arreglo. En esta unidad vas a aprender la metodología que usan los administradores profesionales para resolver problemas en producción.

Metodología de Troubleshooting

El Proceso de Diagnóstico

┌─────────────────────────────────────────────────────────────────┐
│              METODOLOGÍA DE TROUBLESHOOTING                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. DEFINIR EL PROBLEMA                                      │
│      - ¿Qué falla exactamente?                                 │
│      - ¿Desde cuándo?                                          │
│      - ¿Afecta a todos o solo algunos?                        │
│                                                                  │
│   2. RECOLECTAR EVIDENCIAS                                    │
│      - Logs, métricas, configuración actual                     │
│      - Estado de servicios, red, recursos                      │
│                                                                  │
│   3. FORMULAR HIPÓTESIS                                       │
│      - ¿Cuál es la causa más probable?                        │
│      - Ordenar por probabilidad                                │
│                                                                  │
│   4. PROBAR Y VALIDAR                                        │
│      - Un cambio a la vez                                     │
│      - Verificar antes/después                                │
│                                                                  │
│   5. DOCUMENTAR Y PREVENIR                                   │
│      - Registrar qué se hizo y por qué                         │
│      - Automatizar monitoreo para detección temprana           │
└─────────────────────────────────────────────────────────────────┘

Herramientas de Diagnóstico

Comandos Esenciales

BASH
# Procesos y recursos
$ top                    # Monitor interactivo
$ htop                  # top mejorado (más visual)
$ atop                  # top con historial
$ ps aux | head -20   # Procesos activos

# Red
$ ss -tuln             # Puertos en escucha
$ ss -s               # Resumen de conexiones
$ ip -s link          # Estadísticas de interfaz
$ ip route            # Tabla de rutas
$ dig example.com      # Consulta DNS
$ ping -c 4 8.8.8.8  # Conectividad básica

# Sistema de archivos
$ iostat -x 1 5      # I/O stats
$ iotop               # Procesos usando I/O
$ lsof                # Archivos abiertos

# Memoria
$ vmstat 1           # Estadísticas de memoria virtual
$ slabtop             # Slab allocator info

Diagnóstico de Problemas Comunes

BASH
# SERVER NO RESPONDE
# 1. Verificar si está alive
$ ping -c 4 192.168.1.100

# 2. Ver si responde a SSH
$ ssh -o ConnectTimeout=5 user@192.168.1.100 "echo ok"

# 3. Verificar desde consola/IPMI/iLO
$ ipmitool -I lanplus -H 192.168.1.100 -U admin status

# 4. Si es VM, verificar desde hypervisor
$ virsh list
$ virsh console vm-name
BASH
# SERVICIO NO INICIA
# 1. Ver estado
$ sudo systemctl status nginx --no-pager

# 2. Ver logs recientes
$ sudo journalctl -u nginx -n 50 --no-pager

# 3. Ver configuración
$ sudo nginx -t

# 4. Ver si hay otro proceso en el puerto
$ sudo ss -tlnp | grep :80
BASH
# BASE DE DATOS LENTA
# 1. Ver conexiones
$ mariadb -e "SHOW PROCESSLIST;"

# 2. Ver variables
$ mariadb -e "SHOW VARIABLES LIKE 'max_connections';"

# 3. Ver estado de tablas
$ mariadb -e "SHOW TABLE STATUS FROM appdb;"

# 4. Queries lentos
$ mariadb -e "SHOW GLOBAL STATUS LIKE 'Slow_queries';"

Diagnóstico por Área

1. Problemas de CPU

BASH
# Ver carga
$ uptime
14:30:00 up 45 days, load average: 8.52, 2.58, 0.61

# Procesos que más CPU consumen
$ ps aux --sort=-%cpu | head -10

# Detalle por core
$ mpstat -P ALL 1 5

# Ver si es spike o sostenido
$ sar -q 1 10

2. Problemas de Memoria

BASH
# Estado de memoria
$ free -h

# Ver uso por proceso
$ ps aux --sort=-%mem | head -10

# Ver OOM kills recientes
$ dmesg | grep -i "out of memory"
$ journalctl -k | grep -i "out of memory"

# Ver swap activity
$ vmstat 1
# Si si y so están altos constantemente = problemas de memoria

3. Problemas de Disco

BASH
# Espacio disponible
$ df -h

# I/O wait
$ iostat -x 1 5

# Procesos escribiendo/leyendo
$ iotop -o

# Ver inode usage
$ df -i

# Ver mayores consumidores de disco
$ du -sh /* 2>/dev/null | sort -rh | head -10

4. Problemas de Red

BASH
# Ver conexiones establecidas
$ ss -tunapl | grep ESTAB

# Ver errores en interfaces
$ ip -s link

# Ver tabla de rutas
$ ip route

# Ver resolución DNS
$ getent hosts example.com

# Test de conectividad
$ nc -zv host 22
$ curl -I https://example.com

Logs: El Diagnóstico

journalctl: Logs del Sistema

BASH
# Ver todo el sistema (último boot)
$ sudo journalctl -b --no-pager

# Ver servicio específico
$ sudo journalctl -u nginx --no-pager

# Ver desde tiempo específico
$ sudo journalctl --since "2026-03-31 10:00:00" --until "2026-03-31 10:30:00"

# Ver por nivel de error
$ sudo journalctl -p err --no-pager

# Seguir en tiempo real
$ sudo journalctl -f

# Ver kernel messages
$ sudo journalctl -k --no-pager

Logs de Aplicaciones

BASH
# Apache/Nginx
$ sudo tail -f /var/log/apache2/error.log
$ sudo tail -f /var/log/nginx/error.log

# MariaDB
$ sudo tail -f /var/log/mysql/error.log

# Application logs
$ sudo tail -f /var/www/myapp/storage/logs/laravel.log

Analizar Logs Eficientemente

BASH
# Buscar errores
$ sudo grep -i "error\|critical\|alert" /var/log/syslog

# Ver último error
$ sudo tail -100 /var/log/syslog | grep -i error

# Contar errores por tipo
$ sudo awk '/error/ {print $5}' /var/log/syslog | sort | uniq -c | sort -rn

# Buscar en todos los logs
$ sudo grep -r "error" /var/log/* 2>/dev/null | head -20

Casos de Estudio

Caso 1: Servidor Lento, Alta Carga

Síntomas:

BASH
$ uptime
 14:30:00 up 45 days,  load average: 25.52, 18.58, 12.61

$ top
# 16 procesos de "php-fpm" usando 100% CPU cada uno

Diagnóstico:

BASH
# 1. Ver qué proceso consume más
$ ps aux --sort=-%cpu | head -10

# 2. Ver conexiones de red
$ ss -tunapl | grep ESTAB | wc -l
# 5000 conexiones establecidas - ataque?

# 3. Ver logs de aplicación
$ tail /var/log/nginx/error.log
# lots of "upstream timed out" errors

Solución:

BASH
# Identificar: PHP-FPM con loops infinitos
# 1. Aumentar workers
$ sudo sed -i 's/pm.max_children = 5/pm.max_children = 50/' /etc/php/8.1/fpm/pool.d/www.conf

# 2. Recargar
$ sudo systemctl reload php8.1-fpm

# 3. Monitorear
$ watch -n 1 'ps aux | grep php-fpm | wc -l'

Caso 2: No Puedo Conectar a MySQL

Síntomas:

BASH
$ mariadb -u appuser -p
ERROR 2003 (HY000): Can't connect to MySQL server on 'localhost' (111)

Diagnóstico:

BASH
# 1. Ver si el servicio está corriendo
$ systemctl status mariadb

# 2. Ver si está escuchando
$ ss -tlnp | grep 3306

# 3. Ver logs
$ journalctl -u mariadb -n 50

Solución:

BASH
# Si no está corriendo:
$ sudo systemctl start mariadb
$ sudo systemctl enable mariadb

# Si está corriendo pero no en red:
$ sudo sed -i 's/^bind-address.*/bind-address = 0.0.0.0/' /etc/mysql/mariadb.conf.d/50-server.cnf
$ sudo systemctl restart mariadb

Caso 3: Disco Lleno

Síntomas:

BASH
$ df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1       100G  100G     0 100% /

Diagnóstico:

BASH
# 1. Ver mayores consumidores
$ sudo du -sh /* 2>/dev/null | sort -rh | head -10

# 2. Ver logs grandes
$ sudo find /var/log -type f -size +100M -exec ls -lh {} \;

# 3. Ver temporales
$ sudo du -sh /tmp

Solución:

BASH
# 1. Limpiar logs antiguos
$ sudo journalctl --vacuum-time=7d

# 2. Eliminar paquetes antiguos
$ sudo apt autoremove

# 3. Limpiar /tmp
$ sudo rm -rf /tmp/*

# 4. Verificar espacio
$ df -h

Caso 4: DNS No Resuelve

Síntomas:

BASH
$ ping google.com
ping: google.com: Name or service not known

Diagnóstico:

BASH
# 1. Ver configuración DNS
$ cat /etc/resolv.conf

# 2. Probar DNS específico
$ nslookup google.com 8.8.8.8

# 3. Ver si es problema de red
$ ping -c 2 8.8.8.8

Solución:

BASH
# Agregar DNS temporal
$ echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf

# O editar resolv.conf
$ sudo nano /etc/resolv.conf
# Agregar:
# nameserver 8.8.8.8
# nameserver 8.8.4.4

Herramientas Avanzadas

strace: Seguimiento de Syscalls

BASH
# Ver qué hace un proceso
$ sudo strace -p 1234 -f -e trace=network

# Ver qué archivos abre
$ sudo strace -p 1234 -e trace=open,close

lsof: Archivos Abiertos

BASH
# Ver qué tiene abierto un proceso
$ sudo lsof -p 1234

# Ver quién usa un archivo
$ sudo lsof /var/log/syslog

# Ver conexiones de red
$ sudo lsof -i :80

netstat/ss: Estadísticas de Red

BASH
# Ver conexiones
$ ss -tunapl

# Ver estadísticas
$ ss -s

# Ver tabla de rutas
$ ip route

# Ver neighbor table
$ ip neigh

Resumen

Área Comandos Clave Propósito
CPU top, htop, mpstat Ver carga y procesos
Memoria free, vmstat, dmesg Diagnóstico de memoria
Disco df, du, iotop Espacio y I/O
Red ss, ip, ping, dig Conectividad
Logs journalctl, tail, grep Diagnóstico
Syscalls strace, lsof Debug avanzado
